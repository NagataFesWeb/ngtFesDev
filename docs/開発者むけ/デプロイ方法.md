# Vercel デプロイ・運用完全マニュアル

## 1. プロジェクトの初期設定 (Vercel CLI)
ターミナルでプロジェクトのルートディレクトリに移動し、以下のコマンドで開始します。

$ vercel

### 対話型プロンプトの回答指針:
- Set up and deploy? [y/N]: y
- Which scope?: (自分のアカウントを選択)
- Link to existing project? [y/N]: N (新規作成の場合)
- What’s your project’s name?: (プロジェクト名を入力)
- In which directory is your code located?: ./
- Want to modify settings? [y/N]: N
  - Next.jsやViteの設定はVercelが自動判別するため、基本は N でOK。
- Detected a repository. Connect it to this project? [y/N]: n
  - 手動制御（CLIデプロイ）を優先する場合は n を選択。

---

## 2. 環境変数 (Environment Variables) の設定
Next.jsやSupabaseを使用する場合、ビルド前に環境変数の登録が必須です。

### 登録コマンド:
$ vercel env add [変数名]

### 主な変数の設定例:
1. NEXT_PUBLIC_SUPABASE_URL
   - Value: (SupabaseのURL)
   - Mark as sensitive?: N
   - Environments: Production, Preview, Development すべて選択
2. NEXT_PUBLIC_SUPABASE_ANON_KEY
   - Value: (Anon Key)
   - Mark as sensitive?: N (公開前提のため)
   - Environments: すべて選択
3. SUPABASE_SERVICE_ROLE_KEY (管理者用)
   - Value: (Service Role Key)
   - Mark as sensitive?: y (極めて重要)
   - Environments: Production, Preview のみ (Developmentは外す)

---

## 3. 本番デプロイの手順
環境変数の設定が終わったら、以下の2ステップで公開します。

### ステップ1: ローカルでのビルド確認
Next.jsのビルドエラー（Suspenseの欠如など）がないか事前に確認します。
$ npm run build

### ステップ2: Vercelへのデプロイ
Vercelのサーバー側でビルドを実行させる最も標準的な方法です。
$ vercel --prod

---

## 4. トラブルシューティング
ビルドエラーが発生した場合のチェックリスト：

- エラーログの確認:
  `Inspect: https://vercel.com/...` のURLをブラウザで開き、Logsを確認する。
- Suspense Boundary エラー:
  `useSearchParams()` を使用しているコンポーネントは、必ず `<Suspense>` で囲む。
- 環境変数の反映:
  環境変数を追加・変更した後は、再度 `vercel --prod` を実行して再ビルドが必要。

---

## 5. 負荷テスト (k6) との連携
本番環境 (Production URL) に対してテストを行う際の注意点：

- ターゲットURLの変更:
  load-test.js 内の BASE_URL を発行された `https://[project-name].vercel.app` に書き換える。
- 実行コマンド:
  $ k6 run load-test.js
- 制限の確認:
  Vercelダッシュボードの 「Settings > Usage」 から、Bandwidth(100GB/月) や 
  Serverless Function Execution の残り枠を定期的に確認する。
